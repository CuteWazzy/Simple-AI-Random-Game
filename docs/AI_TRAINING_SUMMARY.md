----------------------------------------------------------------
一、项目概述
----------------------------------------------------------------

【项目目标】
基于用户提供的 C++ 数字对战游戏代码，分析游戏规则，并训练一个
基于神经网络的 AI 参加这个游戏，通过自博弈 + 遗传算法不断进化。

【游戏简述】
- 4 位玩家，每人初始 HP=10
- 每回合当前玩家选择目标，抽 0~999999 的随机数
- 命中数 = 数字中含 (6,9)/(7,8)/(9,1) 两位子序列的个数
- 5 位数命中翻倍；4 位选道具；3 位回 2 血；2 位回 4 血；1 位 +10 护盾
- 道具：flip（反转数字）/edit（修改某位 ±1）/reroll（重抽）
- 自瞄（攻击自己）：未命中不扣血，命中扣自己血
- 杀光其它玩家者获胜

----------------------------------------------------------------
二、技术架构
----------------------------------------------------------------

【1. 游戏引擎（game.py）】
- Python 版忠实移植自 C++ 代码
- 状态机：CHOOSE_TARGET → ITEM_DECISION → ITEM_CHOICE →
          EDIT_POS → EDIT_DELTA → REWARD_CHOICE → GAME_OVER
- 支持自瞄（攻击自己），自瞄命中扣自己血
- 121 维状态编码（10 玩家 × 9 维 + 全局特征）
- 道具使用追踪：item_hit_delta / item_dmg_avoided /
  item_useless_count / item_self_aim_hurt

【2. 神经网络（model.py）】
- 共享 MLP 主干：input(121) → 128 → 128 → 128
- 6 个动作头：target / item_decision / item_choice /
              edit_pos / edit_delta / reward_choice
- 价值头：标量 V(s)
- 总参数量：52252
- 支持多种激活函数（ReLU/GELU/Tanh/ELU）

【3. 训练算法（train.py）】
- REINFORCE 带价值基线
- 复合奖励 shaping（血量 + 道具 + 道具命中 + 道具防御 +
  无效惩罚 + 反向防御）
- 自博弈数据收集 + 批量训练
- 支持续训（自动从 checkpoint 加载）

【4. 遗传算法（genetic_train.py）】
- 4 种群并行进化（不同激活函数/温度/奖励权重）
- 多进程训练（ProcessPoolExecutor，绕过 GIL）
- 锦标赛评估（跨种群对战）
- 遗传操作：精英保留 + 线性交叉 + 高斯变异
- 支持续训（保留历史最佳网络）

----------------------------------------------------------------
三、奖励函数演进（10 个版本）
----------------------------------------------------------------

【v1-v2 基础版】
  reward = w_hp × ΔHP_self + w_opp × (-ΔHP_opp)
  w_hp=0.05, w_opp=0.03

【v3 加道具奖励】
  + w_item × Δitems + w_hit × hit_via_item + w_def × dmg_avoided
  w_item=0.005, w_hit=0.15, w_def=0.10
  效果：道具使用从 0 → 745 次/30 局

【v4 加无效惩罚】
  + w_useless × useless_via_item  (w_useless=-0.10)
  效果：道具使用从 745 → 62 次（过度抑制）

【v5 加反向防御】
  + w_self_aim × self_aim_hurt  (w_self_aim=0.25)
  修复盲区：防御者 flip 让自瞄者反伤应给奖励

【v6 恢复鼓励】（解决模式坍塌）
  w_useless=0.0, w_item=0.05, w_hit=0.50, w_def=0.30
  eps_greedy=0.15（强制探索）
  效果：道具使用恢复到 237 次/30 局

【v7-v8 遗传算法引入】
  4 种群并行 + 锦标赛 + 遗传操作
  效果：v8 gen10 道具使用 93 次/30 局

【v9-v10 持续进化】
  gen27：道具使用 176 次，胜率 38.9%
  gen110：道具使用 552 次，胜率 55.6%

【v11（当前）加小惩罚】
  w_useless=-0.01 ~ -0.04（按种群调整）
  gen150：道具使用 529 次，胜率 61.1%

【最终奖励公式】
  reward = 0.05  × ΔHP_self           自己 HP 变化
         + 0.03  × (-ΔHP_opp)         对手 HP 减少
         + 0.05  × Δitems             道具净增量
         + 0.50  × hit_via_item       道具导致的额外命中
         + 0.30  × dmg_avoided        道具导致的伤害避免
         + (-0.03) × useless_via_item 无效道具使用扣分
         + 0.40  × self_aim_hurt      反向防御奖励

四、训练历程（11 个版本，150 代进化）
----------------------------------------------------------------

【阶段 1：基础训练（v1-v2，0 → 1480 轮）】
- 单种群 REINFORCE 训练
- 基础奖励函数（血量 + 对手伤害）
- 学会：自瞄保命、集火残血
- 道具使用：0 次（AI 完全不用道具）

【阶段 2：道具奖励引入（v3，1965 轮）】
- 加 w_hit / w_def / w_item
- 道具使用：0 → 745 次/30 局
- 学会：edit 调整数字、reroll 求变、flip 防御

【阶段 3：无效惩罚（v4，2545 轮）】
- 加 w_useless=-0.10
- 道具使用：745 → 62 次（过度抑制）
- 学会：少用道具，但策略单一

【阶段 4：盲区修复（v5，2795 轮）】
- 加 w_self_aim（反向防御奖励）
- 修复防御者 flip 让自瞄者反伤的盲区
- 学会：用 flip 反向防御自瞄流

【阶段 5：模式坍塌修复（v6，4085 轮）】
- w_useless=0, w_item 调高, eps_greedy=0.15
- 道具使用：38 → 237 次（恢复）
- 学会：重新使用道具，4 位奖励选 reroll

【阶段 6：遗传算法引入（v7-v8，gen1-10）】
- 4 种群并行（ReLU/GELU/Tanh/ELU）
- 多线程训练（受 GIL 限制）
- 锦标赛评估 + 遗传操作
- 胜率：33.3%（C_tanh_conservative）

【阶段 7：多进程优化（v9，gen27）】
- ThreadPoolExecutor → ProcessPoolExecutor
- 4.6x 加速（60-90s/代 → 13s/代）
- 胜率：38.9%（A_relu_hot）

【阶段 8：持续进化（v10，gen110）】
- 110 代进化
- 胜率突破 50% → 55.6%
- 道具使用 552 次/30 局
- y率 51.5%（AI 更愿意用道具）

【阶段 9：小惩罚回归（v11，gen150）】
- w_useless=-0.01 ~ -0.04
- 胜率：61.1%（历史最高）
- 道具使用 529 次（略降但更精准）

六、AI 学到的策略
----------------------------------------------------------------

【1. 自瞄策略】
- 残血时自瞄保命（规避 5/6 位+未命中的自扣血风险）
- 自瞄未命中免疫自扣血
- 自瞄命中会反伤自己（双刃剑）

【2. 道具策略】
- flip：反向防御（让自瞄者反伤）、破坏对手命中
- edit：精准调整数字位（创造 (6,9)/(7,8)/(9,1) 命中）
- reroll：攻击者求变（抽到差数字时换新）
- 4 位奖励选择：根据库存选 flip/edit/reroll

【3. 道具组合】
- reroll + edit×N：换数字后精准调整
- edit×N：连续修改某位创造命中
- flip×N：反转数字改变命中数

【4. 位数策略】
- 1 血时追求 4 位数（规避自扣血 + 获得道具）
- 5 位数高风险高回报（命中翻倍）
- 6 位数最常见但无奖励

【5. 集火策略】
- 优先攻击 HP 最低的玩家
- 集火某个玩家形成人数优势
